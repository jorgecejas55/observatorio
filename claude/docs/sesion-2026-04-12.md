### 2. Resolución de Bugs Críticos

#### Bug #1: Error 405 Method Not Allowed en rutas dinámicas

**Problema:** Al intentar editar un registro (PUT), fallaba con error 405

**Causa:** Next.js 15 breaking change — `params` ahora es `Promise` y debe ser awaited

**Solución aplicada:**
```typescript
// ❌ ANTES (Next.js 14)
export async function PUT(req: Request, { params }: { params: { id: string } }) {
  const result = await update(params.id, data)
}

// ✅ AHORA (Next.js 15)
export async function PUT(req: Request, context: { params: Promise<{ id: string }> }) {
  const params = await context.params
  const result = await update(params.id, data)
}
```

**Archivos corregidos:**
- `museo-virgen-valle/ocasionales/[id]/route.ts`
- `museo-virgen-valle/institucionales/[id]/route.ts`
- `museo-adan-quiroga/ocasionales/[id]/route.ts`
- `museo-adan-quiroga/institucionales/[id]/route.ts`

#### Bug #2: Registros históricos sin ID no editables

**Problema:** Los registros anteriores al `setup()` no tenían ID → `editando.id` era `undefined` → PUT fallaba

**Causa:** Columna `id` agregada después de tener miles de registros en los Sheets

**Solución:** Crear función `generarIDsFaltantes()` en GAS

**Implementación:**
```javascript
function generarIDsFaltantes() {
  const ss = SpreadsheetApp.openById(SHEET_ID);
  
  // Para visitas ocasionales
  var wsOcas = ss.getSheetByName(SHEETS.OCASIONALES);
  var data = wsOcas.getDataRange().getValues();
  var idColIndex = data[0].indexOf('id');
  
  for (var i = 1; i < data.length; i++) {
    if (!data[i][idColIndex]) {
      wsOcas.getRange(i + 1, idColIndex + 1).setValue(Utilities.getUuid());
    }
  }
  
  // Para visitas institucionales (mismo proceso)
  // ...
  
  // Invalidar caché
  invalidateCache('ocasionales_list');
  invalidateCache('institucionales_list');
}
```

**Resultado:**
- Virgen del Valle: ~1000+ IDs generados
- Adán Quiroga: ~500+ IDs generados

#### Bug #3: Mensajes de error crípticos

**Problema:** Error "Unexpected end of JSON input" cuando GAS no configurado o falla

**Solución:** Mejorar helper `gasPost()` con validaciones detalladas

**Implementación:**
```typescript
async function gasPost(body: object) {
  if (!GAS) {
    throw new Error('MUSEO_*_SCRIPT_URL no está configurada en .env.local')
  }

  const res = await fetch(GAS, {
    method: 'POST',
    headers: { 'Content-Type': 'text/plain;charset=utf-8' },
    body: JSON.stringify(body),
  })

  if (!res.ok) {
    throw new Error(`Error en GAS: ${res.status} ${res.statusText}`)
  }

  const text = await res.text()
  if (!text) {
    throw new Error('GAS devolvió una respuesta vacía')
  }

  try {
    return JSON.parse(text)
  } catch (err) {
    throw new Error(`GAS devolvió respuesta inválida: ${text.substring(0, 100)}`)
  }
}
```

**Beneficio:** Mensajes claros y accionables para debugging

## 🧠 Lecciones Aprendidas

### Técnicas

1. **Next.js 15 Breaking Change:** Siempre `await params` en rutas dinámicas
2. **Generación de IDs:** Los datos históricos necesitan IDs retroactivos cuando se implementa un sistema CRUD
3. **Debugging con emojis:** Console.logs temporales con 🔍 son fáciles de identificar y remover
4. **Validación de respuestas:** Parsear JSON siempre con try/catch para mensajes de error claros
5. **Testing progresivo:** Primero GET, luego POST, luego PUT/DELETE — aislar problemas

### Proceso

1. **Replicación != Copy/Paste:** Aunque es replicación, hay que verificar compatibilidad (ej: Next.js 15)
2. **Infraestructura primero:** Desplegar GAS y configurar .env ANTES de probar frontend
3. **Datos de prueba:** Crear un registro de prueba ayuda a detectar bugs inmediatamente
4. **Documentación concurrente:** Actualizar docs mientras se codea evita olvidos
