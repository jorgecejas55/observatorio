### 2. Optimizaciones de Rendimiento Implementadas

**Nivel 1 - Google Apps Script:**
- CacheService con duración de 5 minutos
- Keys: `ocasionales_list`, `institucionales_list`
- Invalidación en create/update/delete
- **Resultado:** 10-20x más rápido en consultas repetidas

**Nivel 2 - Next.js API Routes:**
- Caché de 60 segundos (`revalidate: 60`)
- Reduce llamadas al GAS
- **Resultado:** Menos requests al backend

**Nivel 3 - React:**
- `useCallback` en 5 handlers por página
- `useMemo` en 2 cálculos por página
- **Resultado:** UI más fluida, menos re-renders

## 🧠 Lecciones Aprendidas

### Técnicas
1. **Regla de Oro de React:** Hooks siempre ANTES de early returns
2. **Caché en 3 niveles:** GAS + Next.js + React para máximo rendimiento
3. **Invalidación es crítica:** Caché sin invalidación = datos obsoletos
4. **Parseo tardío:** No parsear inputs hasta validación final
5. **Conversión de tipos:** `String()` para comparar valores mixtos (number/string)

### Arquitectura
1. Un contexto de autenticación por museo (evita confusión)
2. Componentes genéricos reutilizables (TablaVisitas, FormVisita*)
3. Biblioteca centralizada para datos geográficos
4. API routes organizadas por funcionalidad

### Optimización
1. Cachear solo lo que vale la pena (lecturas frecuentes)
2. Duración de caché según volatilidad de datos (5min para visitas históricas)
3. useCallback/useMemo cuando hay impacto medible, no por default
