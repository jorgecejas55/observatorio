```typescript
  if (campo === 'Procedencia ') {
    nuevasVisitas[index] = { ...nuevasVisitas[index], [campo]: valor, 'Lugar de procedencia ': '' }

const toggleCanal = (index, canal) => {

  if (canales.includes(canal)) {
    nuevasVisitas[index].canalesSeleccionados = canales.filter(c => c !== canal)
  } else {

  }
```

## 🐛 Problemas Resueltos

### Error: Procedencia no muestra opción seleccionada

**Causa:** 
- Select de procedencia tenía opciones hardcodeadas en lugar de usar `PROCEDENCIAS.map()`
- onChange hacía dos llamadas a `actualizarCampo` cuando el reset ya estaba en la función

**Solución:**
```typescript
// Antes (hardcodeado)
<select>
  <option value="">Seleccionar...</option>
  <option value="Internacional">Internacional</option>
  <option value="Nacional">Nacional</option>
  <option value="Provincial">Provincial</option>
  <option value="Residente">Residente</option>
</select>

// Después (con array)
<select>
  <option value="">Seleccioná una opción</option>
  {PROCEDENCIAS.map(p => (
    <option key={p} value={p}>{p}</option>
  ))}
</select>
```

### Error: Canal difusión como input text en lugar de checkboxes

**Causa:** Formulario de carga masiva usaba `canal_difusion: string` con input text

**Solución:**
- Cambiar interface a `canalesSeleccionados: string[]`
- Implementar `toggleCanal(index, canal)` para manejar checkboxes
- Renderizar checkbox grid 2 columnas idéntico a FormVisitaOcasional
- Al guardar: `canal_difusion: visita.canalesSeleccionados.join(', ')`

### Error: Lugar procedencia sin label dinámico

**Causa:** Label estático "Lugar de procedencia" en lugar de cambiar según tipo

**Solución:**
```typescript
<label>
```

## 🧠 Lecciones Aprendidas

### Consistencia entre formularios
- Formulario individual y carga masiva DEBEN usar mismos campos y componentes
- Hardcodear opciones causa bugs de binding
- Siempre usar arrays de constantes y `.map()` para generar options

### Filtros combinados
- Año + periodo trabajando juntos mejora UX
- No eliminar filtro existente, agregar nuevo filtro complementario
- `useMemo` esencial para performance con filtros múltiples

### Exportación de datos
- XLSX más intuitivo que CSV para usuarios municipales
- Exportar solo datos filtrados, no todo
- Incluir timestamp en nombre de archivo

### Carga masiva
- Array dinámico de visitas con botones agregar/quitar
- Validación antes de guardar para evitar envíos incompletos
- Contador de exitosas/errores para feedback claro
- Reset form solo si todas exitosas
